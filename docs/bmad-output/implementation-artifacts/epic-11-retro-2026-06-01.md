## 2. Qué salió bien

1. **Reutilización de patrón ganadora.** Un único aggregator genérico (`aggregate-binaria-by-serie`) + un runner compartido (`run-binaria-interior`) cubrieron 3 tipos de elección distintos (balotaje histórico, plebiscito con No derivado, referéndum Sí/No explícito). La rebanada 11.1 dejó el patrón probado y 11.2–11.4 fueron variaciones casi mecánicas.
2. **La rebanada vertical primero (consejo del advisor) pagó.** Hacer balotaje-2024 end-to-end ANTES de fanout reveló y resolvió el desbloqueante de niveles-por-elección una sola vez; las demás stories lo heredaron sin tocar UI.
3. **Los gates atajaron bugs reales antes de producción.** El bug de placement >100% (denominador de cobertura sin no-partidarios) y el riesgo de swap de columnas de candidato se detectaron en verificación, no en deploy.
4. **Verificación honesta multinivel.** ETL gates → gate:data/gate:escaleras → astro build → render en browser. Cada capa atrapó algo distinto.

## 3. Qué costó / lecciones

1. **Hipótesis temprana equivocada sobre la fuente de verdad.** Asumí que `elections-meta.json` gobernaba la UI; el real es `src/config/departments.json` (`getStaticPaths`). Costó pasos.
   - **Lección:** verificar quién *consume* una config (grep de imports) antes de teorizar. Registrado en memoria [[fuente-de-verdad-rutas]].
2. **`checkCoverage` tiene una convención de denominador implícita** (cuenta válidos + no-partidarios). Pasarle solo válidos dio placement >100%.
   - **Lección:** al reusar un gate, replicar exactamente el `totalCanonico` que usan los runners existentes (emitido completo), no improvisar.
3. **Caractericé mal el estado de 8.5 en 11.5** ("deuda parcial en progreso") cuando 8.5 estaba `done` con exclusiones deliberadas. Lo corrigió el feedback de Juan.
   - **Lección:** antes de etiquetar algo como "deuda de Epic X", leer el story file de Epic X para confirmar alcance entregado vs. excluido-por-diseño.
4. **Un swap de columnas A/B no lo atrapa ningún gate estructural** (solo voltea el ganador). Hubo que agregar un cross-check contra resultados nacionales conocidos.
   - **Lección:** para datos binarios/candidato, un ancla externa (resultado publicado) es la única defensa contra inversión silenciosa.
